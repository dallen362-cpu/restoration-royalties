# Claims to Verify Before Using With Prospects / Public

These statements appear on TELVERGENCE marketing/demo surfaces. Each is a **regulated or
substantiable claim** — confirm it is true and documented before presenting to prospects,
investors, or the public. Unsubstantiated versions create false-advertising (FTC) and
regulatory exposure.

> Scope note: this repo can only change files under `restoration-royalties`. The live
> claims on **telvergence.com** (and `app.html`) live in the separate `telvergence` repo —
> update them there too.

## High priority (regulated / certifiable)
- [ ] **"HIPAA-ready"** — do you have a HIPAA compliance program + BAAs in place? If not, drop or qualify.
- [ ] **"SOC 2"** — is there a completed SOC 2 report (Type I/II)? If not in progress, remove.
- [ ] **"Zero data breaches" / "zero breaches"** — defensible as written? Prefer "no known breaches to date."
- [ ] **"99.997% uptime"** — backed by monitoring data / an SLA? Cite the measurement window.
- [ ] **"Blockchain-verified call records"** — is this actually implemented, or roadmap? Label accordingly.
- [ ] **"STIR/SHAKEN verified"** — confirm attestation level with the carrier.
- [ ] **"Military-grade / end-to-end encryption"** — specify the actual standard (e.g. TLS 1.3, AES-256) instead of marketing terms.

## Medium priority (factual claims)
- [ ] **"35+ years in telecom"** — whose experience, and is it accurate for the entity?
- [ ] **"30%+ average savings"**, **"340% increase in lead capture"**, **"60% faster response time"** — source data / case study on file? Add "results vary" qualifier.
- [ ] Any **specific MRR / customer / line counts** — the demo uses illustrative numbers; never present them as actuals.

## Recommended global fixes
- Add a footer disclaimer on public marketing: *"Figures are representative; certifications and metrics available on request."*
- Keep the demo's "illustrative sample data" badge (already present).
- Where a capability is planned, label it **"Roadmap"** rather than implying it's live.
